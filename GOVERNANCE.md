# Governance

## Overview

Venture-Ops is currently maintainer-led. The goal is to keep the project practical, founder-friendly, and easy to evolve without introducing a lot of ceremony.

## Decision Making

- **Architecture and workflow decisions**: maintainer-led
- **Bug fixes**: welcome with clear reproduction and narrow scope
- **Documentation improvements**: welcome from anyone
- **New features**: should start with an issue so the project does not accumulate overlapping workflows

## Contributor Path

Anyone can contribute.

- start with bugs, docs, or small workflow improvements
- show care around data boundaries and local-first behavior
- prefer clarity and safety over feature count

As the project grows, contributors with a track record of good judgment can take on more review and maintenance responsibilities.

## Values

- quality over quantity
- founder data is private by default
- human in the loop
- no spam automation
- local-first and transparent workflows
