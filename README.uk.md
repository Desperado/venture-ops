# Venture-Ops

> Операції з залучення інвестицій на базі ШІ, побудовані на Claude Code. Режими навичок для пошуку інвесторів, оновлення pitch-deck, подачі заявок до акселераторів та розкладу підтримки зв'язку. Пакетна обробка, генерація PDF.

*Натхнено [santifer/career-ops](https://github.com/santifer/career-ops) — адаптовано для засновників, що залучають раунди.*

---

Venture-Ops — це командний центр для залучення інвестицій на базі ШІ для засновників: пошук інвесторів, оновлення pitch-deck, складання заявок, відстеження воронки та ринково-обізнані операції з фандрейзингу.

Замість того щоб жонглювати старими матеріалами, розрізненими нотатками про інвесторів і шаблонними заявками, ви отримуєте одну структуровану систему, яка допомагає:

- знаходити інвесторів і програми, що справді підходять вашій компанії
- оновлювати свою історію, коли deck, сайт і продукт розходяться
- помічати, чого бракує, перед дзвінком із засновником або зустріччю з партнерами
- готувати чистіший аутрич, заявки та підтримку зв'язку
- вести живу пам'ять фандрейзингу замість того, щоб починати з нуля щотижня

> **Важливо:** Venture-Ops — це не бот для спаму інвесторів. Це система загострення. Вона допомагає засновникам обирати кращі цілі, стискати історію та готувати кращі матеріали. Засновник все одно вирішує, що відправляти.

## З першого погляду

![Огляд Venture-Ops](assets/readme/venture-ops-overview.png)

Простими словами:

1. Ви надаєте системі поточний deck, сайт, контекст засновника та цілі.
2. Вона порівнює ці вхідні дані, знаходить застаріле або відсутнє і перевіряє, які інвестори підходять.
3. Вона ставить цілеспрямовані уточнюючі запитання там, де історія ще слабка.
4. Вона дає оновлені матеріали, ранжований список цілей і воронку для роботи.

## Що робить цей інструмент

Venture-Ops перетворює будь-який AI-кодинг CLI на операційну систему фандрейзингу на боці засновника.

Розроблено для засновників, яким потрібно:

- залучити pre-seed, seed або ранній інституційний раунд
- подавати заявки до акселераторів, програм для засновників, студій або грантів
- підтримувати актуальність pitch deck, не переписуючи його з нуля щомісяця
- розуміти, які VC справді підходять, а які — марна трата часу
- готувати відповіді на заявки та вступні тексти для конкретних цілей
- вести структуровану пам'ять про трекшн, докази, ризики та заперечення інвесторів

Система є агентною: вона може перевіряти ваш публічний стартап-сайт, порівнювати його з поточним deck, виявляти застарілий контент, пропонувати переконливіші докази та готувати цільові матеріали для наступної розмови.

## Що реально робить засновник

Якщо ви нетехнічний, робочий процес простий:

1. Помістіть вашу поточну історію в `startup.md`, `founder-bio.md` та `traction-digest.md`.
2. Додайте цільовий раунд і географію в `config/profile.yml`.
3. Додайте імена інвесторів або URL в `investors.yml`.
4. Попросіть агента оновити deck, просканувати інвесторів або порівняти цілі.
5. Перегляньте згенерований шортліст і дайте відповіді на уточнюючі питання.
6. Відправляйте лише найкращі заявки та вступи, а не все скрізь.

## Функції

| **Сканер цілей** | Знаходить інвесторів, акселератори, бізнес-ангелів і програми для засновників, що відповідають вашій стадії, сектору, географії та раунду |
| **Оцінка відповідності** | Пояснює, чому ціль підходить або не підходить, замість того щоб просто давати ім'я |
| **Оновлення deck** | Порівнює deck, сайт, вхідні дані засновника та поточні докази для виявлення застарілих тверджень і відсутніх слайдів |
| **Генерація pitch deck** | Складає наратив з 10-12 слайдів і експортує в HTML/PDF |
| **Цикл питань засновника** | Ставить найцінніші запитання, коли ваша історія неповна або суперечлива |
| **Пам'ять новин / трендів** | Відстежує ринкові зрушення, релевантні події та хуки часових категорій у `market-watch.md` |
| **Відстеження воронки** | Ведеться єдине джерело правди для цілей, статусів і часу підтримки зв'язку |

## Приклади використання

### 1. Оновіть застарілий deck перед зустрічами з інвесторами

У вас є:

- PDF-deck кількатижневої давності
- стартап-сайт, що еволюціонував
- оновлення продукту, не відображені в наративі

Використовуйте Venture-Ops для:

- порівняння deck із головною сторінкою
- виявлення відсутніх метрик, застарілого охоплення продукту та старих скриншотів
- постановки цільових питань засновнику
- генерації оновленої чернетки з 12 слайдів

### 2. Знайдіть інвесторів зі справжнім відповідністю замість широких списків

Ви — берлінський pre-seed засновник, що будує інфраструктуру для розробників.

Використовуйте Venture-Ops для:

- одноразового налаштування цільового профілю
- сканування курованого всесвіту інвесторів
- ранжування цілей за відповідністю
- визначення, хто заслуговує часу цього тижня, а кого слід пропустити

### 3. Подавайте заявки до акселераторів без шаблонних відповідей

Ви хочете подати заявку до YC, EF, Antler або іншої програми для засновників.

Використовуйте Venture-Ops для:

- адаптації корпоративної історії під кожну програму
- генерації чіткіших біографій засновника та відповідей на заявки
- виділення місць, де deck не пояснює «чому зараз»

### 4. Тримайте фандрейзинговий наратив актуальним

Ринок змінюється швидко.

Використовуйте Venture-Ops для:

- моніторингу поточних трендів і новин суміжних компаній
- оновлення `market-watch.md`
- виявлення сигналів «це треба змінити в deck зараз» перед наступним дзвінком з інвестором

## Швидкий старт

```bash
git clone https://github.com/Desperado/venture-ops.git
cd venture-ops
npm install
npx playwright install chromium
npm run doctor
```

Потім налаштуйте:

1. Відредагуйте `startup.md` із корпоративним джерелом правди.
2. Відредагуйте `founder-bio.md` із бекграундом засновника та довіреністю.
3. Відредагуйте `traction-digest.md` із метриками, клієнтами та доказами.
4. Відредагуйте `market-watch.md` із трендами, рухами конкурентів і часовими хуками.
5. Відредагуйте `config/profile.yml` для стадії, географії, розміру раунду та таргетингу інвесторів.
6. Відредагуйте `investors.yml`, щоб додати фонди, ангелів, акселератори, гранти або програми для засновників, які ви хочете відстежувати.

## Використання

### Локальні команди

```bash
npm run doctor                  # initialize and verify setup
npm run scan -- --dry-run       # preview matching targets
npm run scan                    # append matches to data/pipeline.md
npm run verify                  # validate tracker/report integrity
npm run deck -- deck.html out.pdf
npm run followup                # surface follow-up candidates from tracker
```

### Всередині AI-кодинг агента

```text
/venture-ops                    -> show menu
/venture-ops scan               -> discover matching targets
/venture-ops evaluate {URL}     -> investor/accelerator fit report
/venture-ops deck {target}      -> tailored pitch deck package
/venture-ops refresh            -> audit deck + website + founder updates
/venture-ops news               -> trends/news monitoring and implications
/venture-ops apply {target}     -> application / outreach assistant
/venture-ops pipeline           -> process data/pipeline.md
/venture-ops tracker            -> status overview
/venture-ops followup           -> cadence and draft follow-ups
/venture-ops compare            -> rank multiple targets
/venture-ops deep               -> deep-dive one target
```

Або просто вставте:

- URL VC
- сторінку акселератора
- ваш поточний deck
- ваш стартап-сайт

і попросіть агента запустити відповідний режим.

## Операційний принцип

Це фільтр, а не машина для спаму.

Невелика кількість розмов з інвесторами з високою відповідністю та великим контекстом перевершує широкий холодний аутрич. Venture-Ops повинен робити засновника гострішим, а не гучнішим.

Він також повинен ставати кращим з часом:

- кожне виправлення засновника загострює профіль
- кожна нова метрика покращує deck
- кожне пропущене питання стає майбутнім пунктом чекліста
- кожна ринкова зміна оновлює наративну пам'ять

## Відмова від відповідальності

**Venture-Ops — це локальний відкритий робочий процес, а не хостингова фандрейзингова платформа.**

Використовуючи його, ви визнаєте:

1. **Ви контролюєте свої дані.** Деталі засновника, нотатки про інвесторів, трекшн і матеріали фандрейзингу залишаються на вашій машині, якщо ви не вирішите надіслати їх постачальнику AI.
2. **Ви контролюєте відправки.** Система складає і рекомендує, але не повинна подавати заявки або відправляти аутрич від вашого імені.
3. **Ви перевіряєте поточні факти.** Партнери інвесторів, дедлайни, розміри чеків і умови програм часто змінюються; поточні твердження слід перевіряти перед використанням.
4. **Без гарантій.** Оцінки відповідності — це рекомендації, а не істина. Інвестори — не детерміновані системи. Використовуйте судження.
